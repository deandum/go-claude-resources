---
description: Write and run tests for the codebase
---

Use the tester agent. Detect project language and existing test framework.
Match the project's testing style — don't introduce new patterns.

Follow the test pyramid: 70-80% unit, 15-25% integration, 5-10% e2e.
Mock at boundaries only. Use the prove-it pattern for bug fixes.
Apply Arrange-Act-Assert. Name tests as behavior specifications.

Run tests with race detection after writing. Fix failures before reporting done.
